Tier 1 — High impact, moderate effort
1. LLM-NPC Dialogue — Replace the static trader/healer menus with open-ended conversations. The NPC has a personality (from people.json), knows the current depth, the player's background, and maybe past interactions. You prompt an LLM: "You are Grom the Blacksmith on depth 3. A Fighter approaches with 25 gold. Generate a shop interaction." The response gets parsed for buy/sell intents. This alone would make NPCs memorable — no other roguelike does this at this granularity.
2. Adaptive Hints / Dungeon Master — An LLM that watches the player's state (low HP, confused, near stairs, found a vault) and injects one-line flavor or hints via the message log. "A cold draft whispers from the north — you sense a large chamber ahead." or "The goblin you killed wore a medallion with a familiar crest...". No gameplay effect, but enormous atmosphere gain. You already have self.message() — just add an async LLM call that triggers on key state transitions.
3. Procedural Item Lore — Each weapon, potion, or scroll gets a 1-sentence generated backstory when first identified. "This crossbow belonged to the warden of Crypt-7. Three notches mark three failed escapes." The game already has an identification system — tack on an optional flavor field populated by LLM at item discovery.
Tier 2 — Gameplay-changing, more work
4. Adaptive Enemy AI — No LLM needed, just tracking. Enemies remember player tactics across encounters on the same floor. If you use ranged a lot, orcs start carrying shields (damage reduction). If you spam poison, enemies develop tick-based resistance. This is lightweight state management — the game already has DungeonEnemy with on-hit effects. Add a floor_memory dict that learns and adjusts enemy spawn parameters.
5. Procedural Quests — On entering a floor, an LLM generates a side objective: "The ghost of Archmage Vex wants his signet ring from the vault on depth 5. Reward: Identify all scrolls." Track completion in the message log. The game already has vault rooms, NPCs, and an Orb objective — extending the objective system is natural.

6. AI Companion — A semi-autonomous ally that accepts high-level commands ("attack nearest", "guard", "search for stairs"). Its behavior runs on a simple utility AI (score each action, pick highest). No LLM needed at runtime. The game already has multi-entity turn scheduling with energy — a companion would slot right in.
Tier 3 — Unique, speculative
7. LLM-Generated Dungeon Themes — Before generating a floor, prompt an LLM for a theme: "Generate a floor theme for depth 4: biome, enemies, traps, and a special mechanic." Then the generator populates with theme-appropriate tiles, monsters, and hazards. This would make each run feel genuinely different.
8. Enemy Negotiation — Instead of always fighting, enemies could parley. A goblin might beg for gold, a skeleton might demand you leave its crypt. Use an LLM to generate the offer and parse the player's typed response. High novelty, high complexity.
For maximum impact with minimum code: I'd start with #1 (LLM NPC dialogue) and #2 (adaptive DM hints) — they reuse existing systems (NPC interaction, message log) and don't require new UI or entity types. You could even support both remote APIs and local models (Ollama) so the game works offline.
